# АУРАЛИ (Шахты) — Дизайн лендинга

## 1. Контекст и цель

Проект переводится на новый бизнес: SPA/массажная студия `АУРАЛИ` в городе Шахты, Ростовская область.
Текущая задача — зафиксировать целевой дизайн перед реализацией нового лендинга.

База решений:
- Бренд: `АУРАЛИ` (кириллица)
- Визуальный стиль: `Neo Classic Signature` (выбран пользователем)
- Композиция hero: `H2` (слева оффер, справа атмосферный визуал)
- Тон коммуникации: гибрид (эмоция + конкретика)
- Подход к реализации: `B` (каркас + приоритетные конверсионные блоки)

## 2. Продуктовые решения

### 2.1 Позиционирование

- Без использования слова «премиальный» в копирайте
- Ощущение статуса передается через дизайн, ритм текста и анимацию
- Единый голос бренда: спокойный, уверенный, персональный

### 2.2 Целевой лендинг (структура блоков)

1. Header + Hero
2. Trust-метрики (социальное доказательство)
3. Услуги с категориями и карточками
4. О мастере (от первого лица)
5. Оффер: скидка 15% на первый визит
6. FAQ (6-8 вопросов)
7. Блок для туристов
8. Контакты и как добраться
9. Финальный CTA
10. Footer
11. Sticky WhatsApp (всегда активен)

### 2.3 Контент-правила

- Все конфликтующие данные из `text.md` нормализуются в единую версию для Шахт
- Для каждой услуги обязательны: `длительность`, `цена`, `краткий эффект`
- Исправление опечаток и дублей (включая «Изысканность»)
- Тексты строятся на ощущении результата, а не на сухом перечислении

## 3. UX и конверсия

### 3.1 Обязательные конверсионные элементы

- Главный CTA в hero: запись в WhatsApp
- Sticky WhatsApp-кнопка (mobile + desktop)
- Блок-оффер: `-15%` на первый визит
- Финальный CTA перед футером

### 3.2 FAQ

Минимальный набор тем:
- что брать с собой
- запись и график
- противопоказания
- способы оплаты
- подарочные сертификаты
- где находится студия

### 3.3 Локальная релевантность

- Явные формулировки под Шахты и Ростовскую область
- Блок для туристического трафика с локальными триггерами

## 4. Визуальная система

### 4.1 Направление

`Neo Classic Signature`:
- статусная типографика
- фирменные рамки и линии
- мягкая светлая база (временная нежная гамма до финального выбора палитры)
- аккуратные акцентные контрасты

### 4.2 Hero-композиция H2

- Левая колонка: оффер + подзаголовок + CTA
- Правая колонка: атмосферный визуальный блок
- Первый экран должен быть одновременно имиджевым и конверсионным

## 5. Техническая архитектура (Next.js 16 / React 19 / TS strict)

### 5.1 Структура файлов

- `app/page.tsx` — сборка лендинга из секций
- `app/layout.tsx` — metadata и JSON-LD
- `components/layout/*` — header/footer/sticky controls
- `components/sections/*` — изолированные секции лендинга
- `data/site-content.ts` — единая нормализованная модель контента
- `data/services.ts` — услуги и категории
- `lib/utils.ts` — `cn()` на `clsx + tailwind-merge`
- `lib/seo.ts` — генераторы structured data и SEO-данных

### 5.2 Типизация

- Только TypeScript strict
- Пропсы через `interface`
- Без `any`
- Named exports (кроме `page.tsx` и `layout.tsx`)

### 5.3 Анимации (GSAP)

- Hero timeline: последовательный reveal ключевых элементов
- Services: stagger reveal карточек
- FAQ: управляемое раскрытие секций
- Sticky WhatsApp: мягкий pulse
- Cleanup через `context.revert()`
- `use client` только для интерактивных компонентов

## 6. SEO и структурированные данные

### 6.1 On-page

- Геозапросы по Шахтам и Ростовской области в естественной форме
- Локальные метаданные для главной и служебных страниц

### 6.2 JSON-LD

- Тип: `DaySpa`
- Поля: name, description, address, telephone, email, openingHours, priceRange, image, url

## 7. Риски и ограничения

- Сырые исходные тексты из `text.md` с конфликтами
- Неподтвержденные цифры trust-блока
- Финальная палитра еще не выбрана
- Риск перегруза анимациями на мобильных

## 8. Снижение рисков

- Единый слой нормализованных данных (`site-content`)
- Явные placeholders для непроверенных цифр
- Анимации с graceful degradation и reduced-motion
- Приоритет реализации: сначала конверсия, потом декоративная полировка

## 9. Критерии готовности MVP

- Единый контент под Шахты без дублирования и конфликтов
- Рабочие секции: Hero, Услуги, FAQ, Контакты, Финальный CTA
- Sticky WhatsApp + оффер `-15%`
- Валидные метаданные и JSON-LD
- Корректная адаптивность на мобильных и десктопе
